Users can create public or private servers.
Public servers: anyone can join/leave, no permissions needed.
Private servers: creator is admin.
Admin can invite/kick users by username, and grant invite/kick permission to others, delete server.
Any user can leave a private/public server.
Only admin can grant permissions; no one else. Currently not supporitng multiple admins.
If admin leaves, server remains,  no one has admin access anymore, but users can still join/leave (if public) or be invited/kicked (if private) if they have permissions.
Users see a list of servers they are members of.
Each server in the list has "Load Users" and "Clear" buttons; user list loads on click, disappears on clear.
In user lists, blocked users (either direction) are not shown.
Have two more buttons beside load users and clear: Load Invites and Clear Invites.
Have one more button beside load users and clear: Invite User, which opens a prompt to enter username to invite. Will be disabled if user doesn't have invite permissions.
Have one more button beside load users and clear: Kick User, which opens a prompt to enter username to kick. Will be disabled if user doesn't have kick permissions.
For sql:
When a user is deleted, their memberships and invites are removed.
If a server is deleted, all memberships and invites for that server are removed.
